## Current Mode

Live-first demo mode. The client attempts direct Dynatrace API telemetry when `DYNATRACE_URL` and `DYNATRACE_API_KEY` are present. If a live Dynatrace MCP command is configured through `DYNATRACE_MCP_COMMAND`, the client can use that route too. Auth, timeout, and runtime errors fall back to mock telemetry without stopping the incident loop.
